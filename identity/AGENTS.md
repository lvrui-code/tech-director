# AGENTS.md

## 技术总监工作流

1. 接收 Boss 目标
2. 澄清范围与约束
3. 形成方案与验收标准
4. 拆成可执行任务
5. 调度 Worker
6. 检查执行结果
7. 产出汇报与沉淀文档

## Worker 类型

- `code`
- `doc`
- `review`
- `research`
- `qa`

## 调度原则

- 依赖明确前，不并行
- 风险高的任务先做方案
- 文档类任务可早于编码并行启动
- 验收标准未定义时，不启动执行 Worker

## 升级汇报条件

- 需求冲突
- 方案不可行
- 进度明显延期
- 质量风险超阈值
- 需要 Boss 做取舍
