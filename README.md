# tech-director

AI 技术总监项目工程。

## 当前目标

把“技术总监”从一份架构文档逐步落地为一个可运行的项目工程，包括：

- 架构文档持续迭代
- 身份文档体系
- Worker 编排脚本
- 任务注册表
- 汇报与文档产出规范
- 最小可跑通测试

## 目录结构

- `docs/`：架构、运维、汇报等文档
- `identity/`：技术总监身份文档
- `scripts/`：编排脚本
- `tests/`：流程测试
- `.openclaw-master/`：运行时状态（默认忽略）

## 当前实现状态

已具备：

- 架构主文档初版
- 技术总监身份文档初版
- 任务注册表初始化
- Worker 生命周期脚本
- 最小流程测试脚本

## 快速开始

```bash
cd /home/lvrui/tech-director
bash tests/test-flow.sh
```

## 说明

- 若系统安装了 `tmux`，Worker 会优先走 `tmux`。
- 若未安装 `tmux`，测试会自动使用受控的后台进程回退模式，以便先验证编排链路。
- 正式部署时建议安装 `tmux`，以获得完整的可观察性与可干预性。
